Branchless Programming and CPU pipeline

As the name suggests, branchless programming means "trying" to eliminate conditional branches in some piece of software. Although the applications of this technique are very niche due to tremendous advancements in branch prediction, it is still used in many cryptographic algorithims to prevent many timing attacks.

To know why branches in code lead to decrease in performance, we must have a deeper look into how the CPU works.

Deeper dive into the CPU (Can skip this if you already know stages of CPU pipeline)

Modern CPU's have the ability to split the processing of a machine instruction into a sequence of independent steps known as stages. Instead of processing one instruction completely before starting the next, the CPU overlaps the execution of multiple instructions. The stages are independently set-up in the CPU pipeline and this technique is known as pipelining in Computer Architecture.

While the modern CPU has a lot of stages, the fundamental model consists of 5 different stages. Each stage completes it's work in one clock cycle. They are :

  1. Instruction Fetch (IF) : The CPU reads the next instruction from the Instruction Cache at the address stored in Program Counter.

  2. Instruction Decode (ID) : The control unit interprets the instruction to determine the machine level operation (e.g. ADD, MUL).

  3. Execute (EX) : The arithmetic logic unit (ALU) performs the actual operation.

  4. Memory (MEM) : If the instruction involves reading from or writing to RAM executed via instructions like LOAD or STORE, the accesses happen here.

  5. Writeback (WB) : The final result of operation is written back to the CPU register file, updating the state of the program.

The pipeline works really well if the instructions flow continuously without interruptions. In practice, all pieces of code induce hazards into the pipeline causing the pipeline to stall, sometimes requiring the flushing of the pipeline.

Branch prediction induces something called a Control hazard. This means the CPU does not know which instruction to fetch next until the current branch instruction is executed. That is, say instruction 3 was a branch instruction, before the clock cycle 5, we can't fetch instruction 4 as we don't know which branch will be taken up until its evaluated.

Hence the CPU must guess (predict the branch). If it guesses wrong, it must flush the instructions currently in the IF and ID stage which are redundant and are wasting the clock cycle.

Now-a-days, branch prediction has gotten extremely good at guessing the next instruction correctly. Many even use hardware level neural network to predict stuff. But obviously all of these things are heuristics and hence there is some need for branchless programming.

Ways to do branchless programming

So now that we know the issues with branch prediction, let's look at different ways to do branchless programming.

  1. Boolean multiplier

    • Branched way :
      if (condition) {
    x = value_a;
} else {
    x = value_b;
}
    • Branchless way :
      x = (value_a * condition) + (value_b * !condition);

  2. The lookup table

    • Branched way :
      if (x == 0) return 10;
if (x == 1) return 20;
// ...
    • Branchless way :
      const int results[] = {10, 20};
return results[x];

  3. Two's complement trick : In 32-bit integers, if you arithmetic shift a number right by 31 bits (n >> 31), the result is:

    • 0x00000000 (0) if the number is positive.

    • 0xFFFFFFFF (-1 or all 1s binary) if the number is negative.

    We can use this "mask" to select data without branching.

    1. Minimum and Maximum
      • Branched way : (For min)
        if (x < y) {
    r = x;
} else {
    r = y;
}
      • Branchless way : (For min)
        // If x < y, then -(x < y) becomes -1 (all 1s). 
// If x >= y, it becomes 0.
int r = y ^ ((x ^ y) & -(x < y));
    2. Absolute value :
      • Branchless way :
        int mask = x >> 31; 

// If x is positive: (x + 0) ^ 0 = x
// If x is negative: (x + -1) ^ -1. In 2's complement, this flips the bits and adds 1.
int abs_x = (x + mask) ^ mask;

  4. Power of 2

    1. Checking if a number is power of 2

      bool isPowerOfTwo = (x > 0) && ((x & (x - 1)) == 0);

    2. Modulo power of 2

      x & 15; // where 15 is 16 - 1

  5. Toggling ASCII uppercase and lowercase

    // To Lowercase, force 6th bit to 1
char c = x | 32; 

// To Uppercase, force 6th bit to 0
char c = x & ~32;

// Toggle Case, flip 6th bit
char c = x ^ 32;
