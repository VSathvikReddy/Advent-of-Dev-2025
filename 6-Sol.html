(Q1) 0.25 + 0.5

#include <iostream>

int f() {
    std::cout << "f called\n";
    return 100;
}

namespace A {
    static int x = f();
}

namespace B {
    int g();
    static int y = g();
}

int B::g() {
    return A::x + 1;
}

int main() {
    std::cout << A::x << " " << B::y << "\n";
}

(a) Will the given program compile or not? If it does, What's the output.

(Ans) The program will compile and give

f called
100 101

(b) Give reasoning for your answer.

(Ans)

Both A::x and B::y are non local namespace scope objects defined in the same translation unit. C++ rules guarantee that non-local static objects in a single translation unit are initialized in the order of their definitions.

A::x is initialized first by calling f() (prints "f called", it returns a value of 100). Then B::y is initialized by calling g(), which reads A::x (already initialized) and returns 101.

int main() then prints A::x and B::y.

(Q2) 0.25 + 0.25

(a) What all attributes of a function class do you think can be accessed by a static member function.

(Ans) What I expected was only static attributes since I mentioned it the doc that the this pointer is not available which implies I can't access any of the non-static attributes either since they're associated with an object.

So basically a static member function can access: all static members (private or public) and static functions, and any free functions or global variables visible in scope. It cannot access non-static members (no this), because there is no object context.

(b) Give a reason as to why static functions should not have access to the this pointer.

(Ans) Static member functions do not have access to this because they can be called without any object.
If a static function were allowed to use this, then calling it as A::sfn() would require a valid object pointer even though no object exists. For example, if the function tried to access this->a, there is no meaningful object for this to refer to.
This contradicts the fundamental property of static members: they belong to the class itself, not to any instance.

(Q3) 0.25 + 0.5

#include <iostream>
#include <string>
#include <cassert>
#include <expected>

using std::cout;
using std::string;
using std::endl;
using std::expected;
using std::unexpected;

namespace A{
    namespace v1{
        int counter = 0;
    }
    inline namespace v2{
        int counter = 0;
    }
}

expected<int,int> old_div_fn(int a, int b) {
    A::v1::counter++;
    if(b == 0) return A::v1::counter;
    return a/b;
}

expected<int,int> div_fn(int a, int b) {
    A::counter++;
    auto val = old_div_fn(a,b);
    if(val && b != 0) {
        return *val;
    }
    else if(val) {
        return *val - A::counter;
    }
    else {
        return unexpected(val.error() + A::counter);
    }
}

int main() {
    auto result = div_fn(3,2);
    if(!result) {
        cout << "Bertolt " << result.error() << endl;
    }
    else {
        cout << "Bertholdt " << *result << endl;
    }
    result = old_div_fn(2,0);
    if(!result) {
        cout << "Burrito " << result.error() << endl;
    }
    else {
        cout << "Boruto " << *result << endl;
    }
    result = div_fn(14,0);
    if(!result) {
        cout << "Burntoast " << result.error() << endl;
    }
    else {
        cout << "Bee Swarm Simulator "<< *result << endl;
    }
}

(a) Will the given program compile or not? If it does, What's the output.

(Ans) It will compile (you don't have to mention this but it'll only compile with the -std=c++23 flag) and give the following output

Bertholdt 1
Boruto 2
Bee Swarm Simulator 1

(b) Give reasoning for your answer.

old_div_fn does not have any code to give std::unexpected output, hence it'll always give a valid output.

Meanwhile, div_fn give has code to give std::unexpected output but can never reach there.

First result does the following:

  • Call div_fn(3,2)
  • Increment A::v2::counter, now its value is 1
  • Call old_div_fn(3,2)
  • Increment A::v1::counter, now its value is 1
  • return 1
  • val in div_fn takes a valid number
  • if condition is satisfied, function returns a valid 1
  • In main else branch is called and Bertholdt 1 is printed.

Second result does the following:

  • Call old_div_fn(2,0)
  • Increment A::v1::counter, now its value is 2
  • since b is 0, return a valid 2
  • In main else branch is called and Boruto 2 is printed.

Third result does the following:

  • Call div_fn(14,0)
  • Increment A::v2::counter, now its value is 2
  • Call old_div_fn(14,0)
  • Increment A::v1::counter, now its value is 3
  • since b is 0, return a valid 3
  • val in div_fn takes a valid number
  • else if condition is satisfied, function returns a valid (3-2) = 1
  • In main else branch is called and Bee Swarm Simulator 1 is printed.
