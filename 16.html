Curiously Recurring Template Pattern - CRTP

Introduction

In C++, Polymorphism usually refers to virtual functions. While conveniant, they often incur runtime costs which can be significant in extremely low latency systems, drivers and my favorite - game engines. To understand what CRTP is, let's take an example of an ImageProcessor interface.

This example was generated by ChatGPT so if you find any inconsistencies, please do DM me. I've made sure that the example is correct and accuately depicts the concept but there might be some mistakes!

Without CRTP you would write the code something like this

class IImageProcessor {
public:
    virtual void process(unsigned char* pixels, int size) = 0;
    virtual ~IImageProcessor() = default; 
};

class SepiaFilter : public IImageProcessor {
public:
    void process(unsigned char* pixels, int size) override {
        // Implementation for Sepia
    }
};

class GrayscaleFilter : public IImageProcessor {
public:
    void process(unsigned char* pixels, int size) override {
        // Implementation for Grayscale
    }
};

// Usage
void applyFilter(IImageProcessor* processor, unsigned char* data, int len) {
    processor->process(data, len); 
}

The costs we pay :

  • VTable lookup : When processor->process() is called, the CPU must fetch the object's vptr, look up the function address in the vtable, and then jump to that address

  • Instruction Cache Misses : Because the destination is unknown until runtime, the CPU struggles to pre-fetch instructions.

  • No Inlining : The compiler cannot inline process() because it doesn't know which version will be called. For small functions like getters/setters, this overhead is massive.

The Solution

Enforce static polymorphism via CRTP. Instead of base class being generic, the base class depends on the derived class via templates.

template <typename Derived>
class StaticImageProcessor {
public:
    void process(unsigned char* pixels, int size) {
        // We cast 'this' to the Derived type in compile time to access its methods.
        static_cast<Derived*>(this)->processImpl(pixels, size);
    }
};

class SepiaFilter : public StaticImageProcessor<SepiaFilter> {
public:
    void processImpl(unsigned char* pixels, int size) {
        // Implementation for Sepia
    }
};

class GrayscaleFilter : public StaticImageProcessor<GrayscaleFilter> {
public:
    void processImpl(unsigned char* pixels, int size) {
        // Implementation for Grayscale
    }
};

template <typename T>
void applyFilter(StaticImageProcessor<T>& processor, unsigned char* data, int len) {
    // Compiler sees processor.process -> static_cast -> T::processImpl
    // It can inline the whole chain.
    processor.process(data, len);
}

But why is this working??

The optimization pattern relies upon the formula

class Derived : public Base<Derived>

Inside Base<Derived>, the this pointer is physically of the type Base* but we know for a fact that the object was instansiated as Derived. Thus static_cast<Derived*>(this) is safe.

Now as the compiler knows what T is at compile time, the compiler can perform all the fancy optimizations it wants to while compiling the program.

Some of the perf improvements we observe are :

  • The compiler generates code that is identical to manually calling the function on the derived object. The Interface wrapper vanishes during compilation.

  • Direct inlining : Due to T being known at compile time

  • Smaller objects : No vptr is stored in the class, this saves 8 bytes on a 64 bit system per class where CRTP is used.

Trade offs

  • No heterogenous containers : You cannot have something like std::vector<StaticImageProcessor*>. Because StaticImageProcessor<Sepia> and StaticImageProcessor<Grayscale> are totally different types, they cannot share a container without a common base.

  • Code bloating : applyFilter<SepiaFilter> and applyFilter<GrayscaleFilter> generate two separate copies of the function in the binary. Virtual functions would use just one function.

Question 2 : Quirks of inheritance

Consider the following C++ code. What will happen once this code is compiled and run(if it compiles successfully).

template <typename Derived>
class Base {
public:
    void interface() {
        static_cast<Derived*>(this)->implementation();
    }
};

class Derived : public Base<Derived> {
public:
    void implementation() {
        // ... heavy work
    }
    ~Derived() { 
        // critical cleanup
    }
};

void run() {
    Base<Derived>* p = new Derived();
    p->interface();
    delete p;
}

int main() {

    c.foo();
