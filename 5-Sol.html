Question 1: Why use std::string_view when std::span exists?

Technically, std::span<const char> and std::string_view have the exact same memory layout. However, they serve different semantic purposes and offer different interfaces.

1. Interface and Semantics

  • String Operations: std::string_view is designed specifically for text. It provides an interface almost identical to std::string with methods like .find(), .starts_with(), .substr() etc. std::span is a generic container view, it only knows about elements and has no concept of text processing.
  • Constructors: std::string_view implicitly converts from const char* and std::string. std::span is stricter to prevent accidental type mismatches.

2. Immutability

  • std::string_view is always read-only. It views the data as const char. You cannot modify characters through a string_view.
  • std::span<char> is mutable. Unless you explicitly write std::span<const char>, a span allows you to modify the underlying characters.

Question 2: Holding up-to the legacies

To understand this, we must look at how std::cout and printf handle string output differently.

1. Memory Layout The variable buffer allocates memory on the heap.

Address:  0x100   0x101   0x102   0x103   0x104   0x105 ...   0x....
Data:     [ I ]   [ _ ]   [ L ]   [ o ]   [ v ]   [ e ] ...   [ \0 ]
Index:      0       1       2       3       4       5

2. View (_view) When we create _view = view.substr(2, 4), we create a struct that looks like this:

  • Pointer: 0x102 (Points to 'L')
  • Size: 4

3. Overloads std::cout is overloaded for string_view. It looks at the pointer (0x102), reads exactly Size (4 here) characters, and stops.

  • Output: Love

4. printf("%s", _view.data())

  • _view.data() returns the raw pointer 0x102.

  • printf with %s is a C-style function. It does not know about the size member of the view. It only knows to stop when you hit a null terminator i.e \0.

Question 3: Const Conundrums

This question is analogous to the behaviour of int* const vs const int*.

1. const std::span<int> s1 (Like int* const)

  • The View (Pointer + Size): Is const. You cannot change where it points or shrink it.
  • The Data (int): Is mutable.
  • s1[0] = 100; is Valid. We are changing the data, not the view.
  • s1 = s1.subspan(1); gives Error (2). We are trying to assign a new value to s1 itself, but s1 is marked const.

2. std::span<const int> s2 (Like const int*)

  • The View (Pointer + Size): Is mutable. You can point it somewhere else.
  • The Data (const int): Is const. You cannot touch the data.
  • s2[0] = 100; gives Error (3). We are trying to write to a read-only integer.
  • s2 = s2.subspan(1); is Valid. We are updating the view to point to the next element.
