Callbacks in C++

A callback is a function you pass to another function so that it can "call it back" at a later time.

Think of it like leaving your phone number at a busy restaurant and you say - "Call me back when the table is available."

  • Your phone number = The Callback Function
  • The Restaurant = The function doing the work (e.g., downloading, processing)
  • The Call = Invoking the callback

Callbacks allow for customizable behavior, asynchronous programming (GUI, networking, authentication), and a clean separation between "doing the work" and "what to do after the work is done."

A world without Callbacks

Without callbacks, a function that performs a task cannot know what the caller wants to do afterward. We are forced to either poll repeatedly (blocking the program / spawning a separate thread which polls continuously) or hard-code behavior in the function preforming the work.

Consider a Downloader class. Without callbacks, we might write:

class Downloader {
public:
    void download(const std::string& url) {
        // ... Some code which downloads stuff from url and stores in some buffer string ...
        std::string data = bufferString; 
      
        // In the downloader, we are hard-coding the functionality to print the data. 
        // What if the caller wanted to upload the data to a server instead? 
        std::cout << "The downloaded data is : " << data << "\n"; 
    }
};

This is tight coupling. The Downloader ideally shouldn't care what happens to the data, it should only care about retrieving it.

The Evolution of C++ Callbacks

C++ provides multiple ways to implement this pattern, ranging from the C-style semantics to Modern C++ idioms.

1. Function Pointers (The C Style)

Fast and lightweight, but cannot (easily) capture variables from the outside scope and cannot easily bind to object member functions.

// Global function, but we need to make serverUrl a global field, server a global "object" to get this working
// They have no business being globals :(
void upload(std::string data) {
    server.post(serverUrl, data);
}

downloader.download(downloadUrl, upload);

2. Functors (The Object-Oriented Style)

A class that overloads operator(). It can hold state (member variables), but it is a pain to write.

struct Uploader {
    std::string serverUrl;
    void operator()(std::string data) const { 
        // ... Some code which connects to the server, 
        // assume all the functions and fields used below are well defined
        server.post(serverUrl, data);
    }
};
// This struct is passed as an object to the processing function and then call the overloaded ()
Uploader uploadObj{serverUrl};
downloader.download(downloadUrl, uploadObj);

3. Lambdas (Our good C++ friend)

Inline, anonymous functions introduced in C++ 11. They are essentially syntactic sugar for Functors but much more readable.

std::string serverUrl;

// We capture 'serverUrl' from the local scope to use inside the callback
downloader.download(downloadUrl, [serverUrl](std::string data) {
    server.post(serverUrl, data);
});

4. std::function (The modern way)

The do-it-all of callbacks. It can store a lambda, a function pointer, or a functor. It is very flexible but comes with a slight performance overhead due to type erasure and potential dynamic allocation.

void download(std::string url, std::function<void(std::string)> onFinish) {
    // ... Some code which downloads stuff from url and stores in some buffer string ...
    std::string data = bufferString; 

    onFinish(data);
}

Lambdas can capture variables from their surrounding scope. However, how they modify those captures depends on the keywords used, here mutable

Question 2 : 360 no-scope

Capturing variables by reference [&] is powerful, but it requires careful attention to the scope and lifetime of variables. It is easy to accidentally refer to a variable that is shadowed or no longer exists.

Consider the following scenario where we store a callback for later use.

#include <iostream>
#include <functional>

std::function<void()> globalCb;

void registerCallback(std::function<void()> f) { 
    globalCb = f; 
}

int main() {
    int a = 4;

    registerCallback([&](){ 
        std::cout << a << " "; 
    });

    {
        int a = 99; 
        globalCb();
    }
}

What is the output?

Options
--------------------------
(A) 4
(B) 99
(C) Undefined Behavior
(D) Compilation Error
