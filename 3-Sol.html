Q1 (0.25 + 0.25)

#include <iostream>

constexpr int f(int x) {
    if(std::is_constant_evaluated()) {
        return 2*x;
    }
    std::cout << "test" << std::endl;
    return x;
}

int main() {
    const int x = 1;
    constexpr int b = x + 4;
    constexpr int a = f(1) + (4 * x);
    static_assert(a == b);
}

Q) What will happen when you compile this? Give reasoning.

A) The given program will not compile

One thing to note here is that the const int x = 1; line will result in a compile time constant and not a runtime constant!!

So what else could cause the compilation error?

Well f(1) would evaluate to 2 :)

Hence a = 6 and b = 5 and the static_assert would fail.

      |                   ~~^~~~

Q2 (0.25 + 0.25)

#include <iostream>

constexpr int f(int x) {
    if(std::is_constant_evaluated()) {
        return x;
    }
    std::cout << "test" << std::endl;
    return x;
}

int main() {
    int n = 1;
    const int x = n;
    constexpr int b = x + 4;
    constexpr int a = f(1) + (4 * x);
    static_assert(a == b);
}

Q) What will happen when you compile this? Give reasoning.

A) This program will not compile

Here, the value of x is intialised by something that is not a compile time constant.

int n = 1 is a runtime variable and hence any const expression using it will also become a runtime constant rather than a compile time constant.

So the chain goes as follows:

  • x is not a compile time constant
  • b is not a compile time constant (uses x during initialisation)
  • static_assert doesnt not have a compile time constant expression in it.

Q3 (0.25 + 0.25)

#include <iostream>

constexpr int f(int x) {
    if(std::is_constant_evaluated()) {
        return x;
    }
    std::cout << "test" << std::endl;
    return x;
}

int main() {
    int n = 12;
    const int x = 13;
    constexpr int a = x + f(12);
    int b = n + f(13);
    std::cout << a << " " << b << "\n";
}

Q) What is the output? Give reasoning.

A) This program compiles and the output is:

test
25 25

a is expected to be a compile time constant expression, hence the compiler will attempt to evaluate x and f(12) in compile time. Since both can be evaluated at compile time, they are.

Meanwhile, b is a runtime variable and since f is not a consteval function, it also can be evaluated at runtime. Hence we see the "test" output first.

Then the variables a and b taking the value 25 is straightforward.

Q4 (0.25 + 0.25)

constexpr int div(int a, int b) { return a / b; }

int main() {
    static_assert(div(6, 2) == 3);  
    static_assert(div(1, 0) == 0);  
}

Q) What will happen when you compile this? Give reasoning.

A) This program will not compile.

Since int div(int,int) is a constexpr function, it will evaluate inside the static_assert.

The problem lies in the division by zero, that is what causes the error.
